# notes-ai

A CLI tool that reads your notes (`.md`, `.txt`, `.json`) from the current directory — recursively — and lets you ask questions about them using AI (Anthropic or OpenAI). Run it from any folder and it will use that folder as its context.

---

## Build

Requirements: [Go 1.22+](https://go.dev/dl/)

```bash
git clone https://github.com/4rji/notes-ai
cd notes-ai
go build -o notes-ai .
```

Move the binary to your PATH:

```bash
mv notes-ai /usr/local/bin/
```

Cross-compile for other platforms:

```bash
# Linux
GOOS=linux GOARCH=amd64 go build -o notes-ai-linux .

# Windows
GOOS=windows GOARCH=amd64 go build -o notes-ai.exe .
```

## Configuration

Add your API key to `~/.zshrc` (or `~/.bashrc`):

```bash
# Anthropic (default)
export ANTHROPIC_API_KEY="sk-ant-..."

# Or OpenAI
export OPENAI_API_KEY="sk-..."
export NOTES_AI_PROVIDER="openai"
```

Then reload your shell:

```bash
source ~/.zshrc
```

If no key is found, the program prints setup instructions and exits.

### All environment variables

| Variable | Description | Default |
|---|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key (answers) | — |
| `OPENAI_API_KEY` | OpenAI API key (answers and/or embeddings) | — |
| `NOTES_AI_PROVIDER` | `anthropic` or `openai` (for answers) | `anthropic` |
| `NOTES_AI_MODEL` | Answer model name | `claude-sonnet-4-6` / `gpt-4o` |
| `NOTES_AI_EMBED_MODEL` | Embedding model | `text-embedding-3-small` |
| `NOTES_AI_TOP_K` | Relevant chunks sent per question | `8` |
| `NOTES_AI_MAX_CHARS` | Context limit in fallback mode | per provider |
| `NOTES_AI_SYSTEM` | Custom system prompt | built-in default |

## Usage

```bash
# Interactive chat mode
cd ~/my-notes
notes-ai

# Ask a single question and exit
notes-ai ask "what projects do I have pending?"

# Build / update the embeddings index
notes-ai index

# Rebuild the index from scratch
notes-ai index --rebuild

# Override provider or model at runtime
notes-ai --provider openai
notes-ai --model claude-opus-4-8 ask "summarize my tasks"
```

### Interactive mode commands

A command menu is shown automatically when interactive mode starts.

| Command | Description |
|---|---|
| `/index` | Index / update notes (incremental, RAG) |
| `/reindex` | Rebuild the index from scratch |
| `/info` | Show index status / loaded files |
| `/reload` | Reload and sync changes |
| `/limpiar` | Clear conversation history |
| `/ayuda` | Show the command menu |
| `/salir` | Exit |
| `Ctrl+C` | Exit |

## How it works

`notes-ai` has two modes, chosen automatically:

### Smart search (RAG) — when `OPENAI_API_KEY` is set

Instead of sending every note on each question (slow, expensive, hits rate
limits), it does semantic retrieval:

1. **Indexing** — notes are split into chunks and embedded once. The result is
   stored locally in `.notes-ai-index.json`. Re-indexing is **incremental**:
   only new or changed files (detected by content hash) are re-embedded.
2. **Per question** — your question is embedded, the most similar chunks are
   found via cosine similarity (instant, local math), and only the top-K chunks
   (default 8) are sent to the answer model.

This scales to thousands of notes and keeps cost/rate-limit usage low.
Embeddings come from OpenAI (`text-embedding-3-small`) even when answers are
generated by Claude — the two providers are independent.

### Fallback (full context) — when no embeddings key is available

All notes are concatenated and sent as context, truncated to `NOTES_AI_MAX_CHARS`.
Good for small note collections.

In both modes, conversation history is kept across turns (last 20 exchanges) in
interactive mode. Ignored directories: `.git`, `node_modules`, `vendor`,
`.venv`, `__pycache__`, `.cache`. The index file itself is never indexed.

## Project structure

```
notes-ai/
├── main.go            # CLI entry point, flags, subcommands
├── config/
│   └── config.go      # Env var loading, setup instructions
├── reader/
│   └── reader.go      # Recursive file walker + content hashing
├── index/
│   ├── index.go       # Chunking, local index, incremental sync, search
│   └── index_test.go  # Tests for chunking, cosine, incremental sync
├── ai/
│   └── client.go      # HTTP clients: chat (Anthropic/OpenAI) + embeddings
├── chat/
│   └── chat.go        # Interactive loop, single-question, index command
└── go.mod
```

No external SDKs — only one dependency (`readline`) for the interactive prompt. All API calls (chat and embeddings) are made directly over HTTP, and the embeddings index is a plain local JSON file (no database server).
