# notes-ai

Ask questions about your own notes from the terminal. `notes-ai` reads your
`.md`, `.txt`, and `.json` files from the current directory (recursively) and
answers questions about them using AI (Anthropic or OpenAI). Run it from any
folder and it uses that folder as its knowledge base.

---

## What it does

- **Reads your notes** — walks the current directory and subdirectories for
  `.md`, `.txt`, and `.json` files.
- **Answers questions** — chat interactively or ask a one-off question; replies
  are grounded in your notes.
- **Smart retrieval (RAG)** — instead of sending every note on each question, it
  builds a local embeddings index and sends only the most relevant pieces. This
  keeps it fast, cheap, and avoids API rate limits, even with thousands of notes.
- **Shows token usage** — every answer reports the exact tokens sent and
  received, so you always know the cost.
- **Single binary** — written in Go, no runtime or database. Configured entirely
  through environment variables.

## Build

Requirements: [Go 1.22+](https://go.dev/dl/)

```bash
git clone https://github.com/4rji/notes-ai
cd notes-ai
go build -o notes-ai .
```

Move the binary to your PATH:

```bash
mv notes-ai /usr/local/bin/
```

Cross-compile for other platforms:

```bash
# Linux
GOOS=linux GOARCH=amd64 go build -o notes-ai-linux .

# Windows
GOOS=windows GOARCH=amd64 go build -o notes-ai.exe .
```

## Configuration

Add your API key to `~/.zshrc` (or `~/.bashrc`):

```bash
# Anthropic (default, for answers)
export ANTHROPIC_API_KEY="sk-ant-..."

# OpenAI (needed for embeddings / smart search, and optionally for answers)
export OPENAI_API_KEY="sk-..."
```

Then reload your shell:

```bash
source ~/.zshrc
```

If no key is found, the program prints setup instructions and exits.

> **Tip:** You can use Claude for answers and OpenAI only for embeddings at the
> same time — the two providers are independent.

### All environment variables

| Variable | Description | Default |
|---|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key (answers) | — |
| `OPENAI_API_KEY` | OpenAI API key (answers and/or embeddings) | — |
| `NOTES_AI_PROVIDER` | `anthropic` or `openai` (for answers) | `anthropic` |
| `NOTES_AI_MODEL` | Answer model name | `claude-sonnet-4-6` / `gpt-4o` |
| `NOTES_AI_EMBED_MODEL` | Embedding model | `text-embedding-3-small` |
| `NOTES_AI_TOP_K` | Relevant chunks sent per question | `8` |
| `NOTES_AI_MAX_CHARS` | Context limit in fallback mode | per provider |
| `NOTES_AI_SYSTEM` | Custom system prompt | built-in default |

## Usage

```bash
# Interactive chat mode
cd ~/my-notes
notes-ai

# Ask a single question and exit
notes-ai ask "what projects do I have pending?"

# Build / update the embeddings index
notes-ai index

# Rebuild the index from scratch
notes-ai index --rebuild

# Override provider or model at runtime
notes-ai --provider openai
notes-ai --model claude-opus-4-8 ask "summarize my tasks"
```

### Interactive mode commands

A command menu is shown automatically when interactive mode starts.

| Command | Description |
|---|---|
| `/index` | Index / update notes (incremental, RAG) |
| `/reindex` | Rebuild the index from scratch |
| `/info` | Show index status and session token totals |
| `/reload` | Reload and sync changes |
| `/clear` | Clear conversation history |
| `/help` | Show the command menu |
| `/exit` | Quit (`Ctrl+C` also works) |

> Spanish aliases also work: `/limpiar` (`/clear`), `/ayuda` (`/help`),
> `/salir` (`/exit`).

## How it works

`notes-ai` has two modes, chosen automatically:

### Smart search (RAG) — when `OPENAI_API_KEY` is set

Instead of sending every note on each question (slow, expensive, hits rate
limits), it does semantic retrieval:

1. **Indexing** — notes are split into chunks and embedded once. The result is
   stored locally in `.notes-ai-index.json`, next to your notes. Re-indexing is
   **incremental**: only new or changed files (detected by content hash) are
   re-embedded.
2. **Per question** — your question is embedded, the most similar chunks are
   found via cosine similarity (instant, local math), and only the top-K chunks
   (default 8) are sent to the answer model.

This scales to thousands of notes and keeps cost/rate-limit usage low.
Embeddings come from OpenAI (`text-embedding-3-small`) even when answers are
generated by Claude.

### Fallback (full context) — when no embeddings key is available

All notes are concatenated and sent as context, truncated to `NOTES_AI_MAX_CHARS`.
Good for small note collections.

In both modes, conversation history is kept across turns (last 20 exchanges) in
interactive mode. Ignored directories: `.git`, `node_modules`, `vendor`,
`.venv`, `__pycache__`, `.cache`. The index file is never indexed.

## Token usage

After every answer, `notes-ai` prints the **real** token counts reported by the
APIs (not estimates), so you always know exactly what each question costs:

```
  ┄ tokens: 1,240 enviados · 312 recibidos · 14 en búsqueda
```

| Field | Meaning |
|---|---|
| `enviados` (sent) | Input tokens sent to the answer model: system prompt + retrieved chunks + history + your question. This is what counts against rate limits. |
| `recibidos` (received) | Output tokens generated in the answer. |
| `en búsqueda` (search) | Tokens used to embed your question (RAG mode only). |

- `/info` shows the **cumulative totals** for the whole session.
- `notes-ai index` reports the embedding tokens consumed while building the index.

This makes the benefit of RAG visible: with full-context mode a single question
might send 80,000+ tokens, while RAG typically sends only a few thousand —
the relevant chunks instead of every note.

> The counts are pulled straight from each API's `usage` object
> (`input_tokens`/`output_tokens` for Anthropic, `prompt_tokens`/
> `completion_tokens` for OpenAI), so they are exact.

## Project structure

```
notes-ai/
├── main.go            # CLI entry point, flags, subcommands
├── config/
│   └── config.go      # Env var loading, setup instructions
├── reader/
│   └── reader.go      # Recursive file walker + content hashing
├── index/
│   ├── index.go       # Chunking, local index, incremental sync, search
│   └── index_test.go  # Tests for chunking, cosine, incremental sync
├── ai/
│   └── client.go      # HTTP clients: chat (Anthropic/OpenAI) + embeddings
├── chat/
│   └── chat.go        # Interactive loop, single-question, index command
└── go.mod
```

No external SDKs — only one dependency (`readline`) for the interactive prompt.
All API calls (chat and embeddings) are made directly over HTTP, and the
embeddings index is a plain local JSON file (no database server).
